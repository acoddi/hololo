# 프로젝트 문서

이 폴더는 STM32 기반 스마트 식물 자동 관리 및 PC 모니터링 시스템의 설계 기준을 정리하는 문서 모음이다.

현재 저장소는 Qt Quick 초기 템플릿에 가까운 상태이며, 이 문서들은 앞으로 구현을 진행할 때 기준이 되는 목표 구조를 설명한다.

## 문서 목록

- `project-overview.md`
  - 프로젝트 정의, 목표, 시스템 구성, 데이터 흐름, 개발 순서를 정리한다.
- `architecture.md`
  - STM32와 PC 애플리케이션의 역할 분리, PC 앱 내부 계층, 모듈 책임, 권장 폴더 구조를 설명한다.
- `coding-convention.md`
  - Qt/QML + C++ 프로젝트에서 사용할 네이밍 규칙, 계층 책임 규칙, 파일 구성 원칙을 정리한다.

## 권장 읽기 순서

1. `project-overview.md`
2. `architecture.md`
3. `coding-convention.md`

## 문서 운영 원칙

- 기능을 추가하기 전에 먼저 해당 기능이 어느 계층 책임인지 확인한다.
- 새로운 서비스, ViewModel, 통신 규약이 추가되면 관련 문서를 함께 갱신한다.
- 문서는 "현재 구현"과 "목표 구조"가 다를 수 있으므로, 구현이 따라오면 문서도 함께 맞춘다.
